# FocusBoard 演示项目需求

## 功能需求

- 用户必须可以在输入任务标题后点击“添加任务”，页面立即显示新任务。
- 系统必须拒绝空白任务标题，并在输入框下方显示“请输入任务标题”。
- 用户点击任务复选框后，页面必须显示任务完成样式，同时“已完成”统计数字立即更新。
- 用户点击“进行中”筛选后，任务列表必须只显示未完成任务。
- 用户点击“已完成”筛选后，任务列表必须只显示已完成任务。
- 系统必须从 `/api/tasks` 获取初始任务，接口成功时页面显示服务器返回的任务。
- `/api/tasks` 接口必须返回 HTTP 200 和 JSON 任务数组。
- 接口请求失败时，页面必须显示“部分数据未同步”错误横幅，并在控制台记录错误。
- 用户刷新页面后，系统必须从 localStorage 恢复用户新增的任务。

## 非功能需求

- 页面在 375px 宽度下不得出现横向滚动条。
- 所有输入框和筛选按钮必须具有可访问名称。
- 页面首次加载完成后必须显示明确的页面标题和任务列表区域。
